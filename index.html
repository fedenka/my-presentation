Что такое Bridge?

Разделяет абстракцию и реализацию для независимого изменения

Источник: Refactoring.Guru

Какие проблемы решает?

  • Избегает взрывного роста классов
  • Устраняет жесткую привязку абстракции к реализации
  • Позволяет изменять реализацию во время выполнения

Как работает Bridge?

Абстракция содержит ссылку на реализацию

Диаграмма классов

Abstraction - высокоуровневая логика

Implementation - низкоуровневая реализация

Пример на TypeScript


// Implementation
interface Device {
    isEnabled(): boolean;
    enable(): void;
    disable(): void;
}

class Tv implements Device {
    private power = false;

    isEnabled() { return this.power; }
    enable() { this.power = true; }
    disable() { this.power = false; }
}

// Abstraction
class RemoteControl {
    constructor(protected device: Device) {}

    togglePower() {
        this.device.isEnabled()
            ? this.device.disable()
            : this.device.enable();
    }
}

Преимущества

  • Принцип единой ответственности
  • Принцип открытости/закрытости
  • Упрощает поддержку кода
  • Позволяет создавать платформо-независимые программы

Ограничения

  • Усложняет архитектуру
  • Не всегда очевидна необходимость
  • Избыточен для простых случаев

Где применяется?

  • Графические интерфейсы (разные ОС)
  • Драйверы устройств
  • Кросс-платформенные приложения
  • Когда возможны разные реализации одного функционала

Ключевые моменты

  • Разделяет абстракцию и реализацию
  • Снижает связанность компонентов
  • Упрощает расширение функционала
  • Особенно полезен для сложных систем
